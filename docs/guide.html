🧫 Plaque Toolkit — User Guide

Measure bacteriophage plaque size, count, and turbidity from Petri-dish photos. Interactive editor, batch processing, and publication-ready exports — all in one desktop app.

iPhone HEIC ✓ Batch folders ✓ Drag & drop ✓ No coding needed ✓
🚀 Quick Start — 60 seconds
The fastest path from photo to measurement.
  1. 1
    Launch the app

    Double-click ! Open Plaque Toolkit (BEST).bat in the root folder, or open the installed "Plaque Toolkit (full)" from the Start menu. The app opens with a clean window.

    💡 The all-in-one installer at Output\PlaqueToolkitFullSetup.exe includes the Precise engine with no conda needed.
  2. 2
    Open your plate image

    On the Measure tab, click Open image… or drag-and-drop a TIFF, JPEG, PNG, or iPhone HEIC file onto the window. The dish diameter defaults to 100 mm — adjust if your dish is different.

  3. 3
    Pick an engine & detect

    The engine dropdown defaults to Precise if available, otherwise Current. Hit Detect — plaques appear as colored circles.

    ℹ️ Detection takes 5–30 s depending on image size and engine. The window stays responsive.
  4. 4
    Review & correct in the editor

    The canvas editor opens. Click Erase to remove false positives, Add to mark missed plaques, or rubber-band with Detect area to re-scan a region. Zoom with the scroll wheel.

  5. 5
    Export your data

    Click the bright green ⬇ Export ▼ button to download a CSV, annotated figure, or side-by-side comparison. Results are saved to an out\ folder next to your image.

    ✅ Done! The CSV contains per-plaque area, diameter (mm & px), brightness, and source.

📦 Batch Tab
Process an entire folder of plate images in one click. No editing — fully automatic, with per-plate and pooled output.
  1. 1
    Click the Batch tab

    It is the second tab in the main window.

  2. 2
    Choose your folder

    Click Browse… and select the folder containing your plate images. Supported formats: TIFF, JPEG, PNG, HEIC.

  3. 3
    Set options

    Set the dish diameter (default 100 mm), engine, and whether to use small-plaque mode. A progress bar shows image-by-image status during the run.

  4. 4
    Click Run Batch

    The app processes every image in the folder. If one image fails, it logs the error and continues with the rest.

  5. 5
    Review outputs

    A summary table appears. Click Open output folder to find per_plate.csv, all_plaques.csv, and annotated images for each plate.

💡 Batch does not open the editor. If you need to hand-correct, use the Measure tab for individual plates. Batch is for automated pipelines where you trust the engine.
📊 Compare Turbidity Tab
Cross-phage turbidity comparison using transmitted-light optical density. This is the publishable turbidity measurement.
ℹ️ Turbidity vs size: The Measure tab gives a relative turbidity index per plate (not cross-plate comparable). This tab gives absolute OD values anchored to a shared clear reference — the cross-phage measurement you can actually compare and publish.

Setup your image folder

Name each plate image after its phage: T4.heic, T7.tif, etc. For replicates: T4_1.heic, T4_2.heic — then tick "Group by prefix" in the app.

Optional reference images (recommended)

Blank agar (I₀)
A plate with clear agar and no bacteria — the "100% transmittance" reference. Required for absolute OD. Without it, you get relative transmittance only.
Flat field
Bare light box with no plate — corrects uneven illumination. Reduces artifacts from gradient lighting (iPhone photos especially benefit).
Dark frame
Lens covered, same exposure — removes the camera black-level offset. Optional but helps absolute OD accuracy.

Output files

FileContents
per_phage.csvThe comparison table — per phage: N plates, N plaques, mean/SD of OD, turbidity, diameter. Use PLATE_MEAN columns for statistics.
plaques_all.csvEvery plaque with PHAGE, TRANSMITTANCE, OD, TURBIDITY, CLARITY columns.
qc.csvPer-plate QC: illumination CV, POLARITY_OK, fraction saturated pixels. Check this first.
compare_*.pngBox plots of diameter and turbidity per phage (publication figures).
run_metadata.jsonProvenance: tool version, all settings, image hashes — keep this for Methods.
⚠️ Check qc.csv — if POLARITY_OK=False, your imaging is not transmitted-light (clear plaques should be brighter than the lawn). If ILLUM_CV is high, add a flat-field image.
✅ Validate Tab
Score engine performance against your own hand-corrected ground truth. This is how you generate your own validation data before publishing.

The validation workflow

  1. 1
    Correct some plates by hand

    Use the Measure tab editor to review detections. Remove false positives, add missed plaques, then export: Export ▼ → Ground-truth labels. This saves a labels_name.json file with your verified plaque set.

  2. 2
    Open the Validate tab

    Browse to the folder containing your labels_*.json files. Tick which engines to compare (Current, Precise, etc.).

  3. 3
    Click Run validation

    Each engine is re-run on the original images and scored against your ground truth labels.

  4. 4
    Read the results

    Two tables appear: per-mode (pooled scores across all plates) and per-plate (individual plate breakdown).

    MetricMeaning
    PrecisionFraction of detections that are real plaques (1.0 = no false positives)
    RecallFraction of real plaques that were detected (1.0 = none missed)
    F1Harmonic mean of precision and recall — the overall score to optimize
    Size MAE (mm)Mean absolute error of diameter vs your labels
    Size bias (mm)Systematic over- or under-sizing
    Size Pearson rCorrelation between engine diameters and your labels (1.0 = perfect)
💡 The more plates you hand-label, the more reliable the validation. Aim for at least 3–5 plates per phage, covering sparse and dense plates.
ℹ️ Click "Validation guide" on the Validate tab to open the full publication-grade protocol (docs/VALIDATION_GUIDE.md).
⚙️ Choosing an Engine
Four detection modes — same measurement math, different detectors and validation status.
Published  Peer-reviewed
The exact Trofimova & Jaschke (2021) algorithm, unchanged. The only citable mode for publications. Automatically disables watershed and sensitive settings. Reproduces the literature within ~0.04 mm.

Use for: any numbers that go into a paper.
Current  Default
Same algorithm with bug-fixes: non-truncated numeric values and a corrected dish calibration (uses the roundest large contour, not just the biggest, to prevent the dark surround from hijacking the mm scale).

Use for: routine day-to-day measurement.
Sensitive  More recall
Lowers the size gates to catch sub-0.4 mm plaques. Finds more plaques — but also more false positives (bubbles, glare, lawn texture). Not independently validated.

Use for: exploratory counts, tiny-plaque phenotypes. Always verify by eye.
Precise  Best detector
Fuses PST + PlaqSeg YOLO segmentation. Best accuracy on dense plates where plaques touch or the lawn is textured. Slower (~15–60 s). Not independently validated.

Use for: dense, difficult plates. Also the app default when PyTorch is available.
⚠️ Only Published is citable in a paper. Current, Sensitive, and Precise are in-house extensions that must be validated on your own plates before publishing (use the Validate tab).

Mode interactions

SettingNotes
Published modeForces watershed OFF and sensitive OFF. A citable run can never be altered by extensions.
Split touching plaques (watershed)Works with Current and Sensitive. Splits merged plaques using distance-transform watershed. Apply uniformly across all compared groups.
Precise classifier gateResNet-18 plaque-vs-texture gate, ON by default in the app. Improves precision on textured plates. Not human-validated — treat as a quality filter, not a citable step.
🎯 Precise Engine — How It Works
The most powerful detector in the toolkit. A 9-stage pipeline fusing the validated PST detector with a trained YOLO segmentation model.
  1. 1
    PST front — dish geometry

    Runs the validated PST detector on the original image to find the dish boundary, compute mm/px calibration, and get PST-sensitive plaque centers as recall candidates.

  2. 2
    Artifact masks

    Three masks are built from the dish geometry: (1) lawn ROI = inner 80% of dish radius, (2) blue label mask (colored sticker), (3) hard dish boundary. Any detection on the rim, label, or outside the dish is dropped.

  3. 3
    PlaqSeg YOLO primary detection

    A YOLO segmentation model (tiled at 1280 px) detects plaques on the original image. This is the primary detector. Green circles = PlaqSeg detections.

  4. 4
    Density switch

    If PlaqSeg found a dense plate (PlaqSeg count ≥ 1.5× PST-sensitive count), accept PlaqSeg as final and skip the recall passes — they only add noise on dense plates.

  5. 5
    Gated PST recall (sparse plates)

    For sparse plates, PST-sensitive centers are accepted if: inside lawn ROI, not already near a PlaqSeg hit, and center-vs-ring contrast > 0.030. Orange circles = PST recall additions.

  6. 6–7
    Optional blob recovery + classifier gate

    Blob recovery (off by default) catches any remaining faint spots. The ResNet-18 classifier (on by default in the app) re-scores every candidate and drops texture false positives.

  7. 8
    Union + dedup + output

    All accepted detections are merged, deduplicated, sized, and written to CSV + colored overlay + JSON summary. If the two primary detectors disagree by >50%, an "Uncertainty" flag appears in the app.

ℹ️ Precise runs entirely in-process in the Full installer (no conda needed). In the source/light versions it calls a second plaqseg conda env.

🔬 Cropped Plates for Fiji / ImageJ (scale bars)
Export a cropped plate that carries its mm scale, so you can crop individual plaques in Fiji and add consistent scale bars — without setting the scale by hand each time.
ℹ️ The idea: show the whole plate, then in Fiji crop a plaque and zoom in with a correct mm scale bar next to it. Because the exported TIFF is pre-calibrated, every plaque you crop from it inherits the right scale automatically.

How it works

From the editor's ⬇ Export ▼ → Cropped plate for Fiji (calibrated TIFF), the app crops the dish region and saves a TIFF that embeds the spatial calibration in millimetres, in the exact tag layout ImageJ/Fiji reads automatically (an ImageJ…unit=mm description plus XResolution = pixels-per-mm). A sidecar <name>.fiji.txt records the exact numbers and the Fiji steps.

Doing it for many photos

On the Batch tab, tick "Also export Fiji-calibrated plate crops". Every image in the folder then gets its own calibrated <name>_plate.tif + .fiji.txt in its output sub-folder — the same workflow applied consistently across the whole set.

⚠️ One scale per photo. Each plate's mm/px depends on how big the dish appeared in that specific photo, so the calibration is image-specific. In Fiji's Set Scale, only tick "Global" for crops of the same plate — never reuse one scale across different photos. (Each pre-calibrated TIFF already carries its own correct value, so you normally do not need Set Scale at all.)
💡 Keep the crops clean: this TIFF has no overlays burned in — just the raw, calibrated pixels — so your Fiji figures look exactly how you arrange them. For a labelled whole-plate panel with the app's own scale bar, use Export → Annotated figure instead.

🔬 Calibration
How pixel measurements are converted to millimetres — and what can go wrong.

The formula

The mm scale comes entirely from the detected dish:

mm_per_px = dish_diameter_mm / dish_diameter_px
DIAMETER_MM = 2 * sqrt(AREA_PXL * mm_per_px^2 / pi)

The dish is detected as the roundest large contour in the image (Current/Precise modes). This prevents the dark surround common in top-lit iPhone photos from hijacking the mm scale — a real failure mode of the original algorithm on non-backlit images.

Tilt warning

If the dish looks elliptical in the photo (tilted camera), every mm value is biased. The app measures the dish axis ratio (major/minor) and warns when it exceeds 1.03. Image plates straight-on to avoid this.

What the dish label shows

The orange circle on the canvas is labeled, e.g.:

dish Ø=2754 px · axis 1.00 · 0.0363 mm/px
  • Ø=2754 px — detected dish diameter in pixels
  • axis 1.00 — tilt ratio (1.00 = perfectly circular; warn at >1.03)
  • 0.0363 mm/px — the mm-per-pixel scale used for all measurements on this plate
⚠️ If no dish is detected (whole-dish not in frame, or the background color is similar to the dish rim), the app falls back to pixel-only output and warns. Make sure the entire dish rim is visible.
🌊 Understanding Turbidity
Two different turbidity numbers exist in the toolkit. Use the right one for your question.

TURBIDITY_REL — within-plate only

This appears in every size CSV. It is a within-plate relative index: 0 = the clearest plaque on this plate, 1 = the bacterial lawn.

⚠️ Do NOT compare TURBIDITY_REL across different plates or phages. Each plate is scaled to its own plaques, so a 0.3 on plate A is not the same as a 0.3 on plate B.

Good for: ranking clear-vs-turbid within one plate, QC checks, preliminary screening.

OD / TURBIDITY — cross-plate comparable

This comes from the Compare turbidity tab. Optical density OD = −log₁₀(I/I₀), anchored to a shared clear-agar reference (I₀). This is comparable across plates and phages — the publishable measure.

💡 Provide a blank agar image to get absolute OD. Without it you get only relative transmittance.
ℹ️ For true absorbance (publishable OD), use RAW or linear TIFF input with dark and flat corrections. iPhone HEIC/JPEG is tone-mapped, so OD is an apparent OD — a within-session relative measure. State this in your methods.

Good for: cross-phage turbidity comparison, titer calculation, the real publishable number.

📰 Publishing Results
What to cite, what to report, and what caveats to state in your methods.

Citation

Trofimova E, Jaschke PR. Plaque Size Tool: An automated plaque analysis tool for simplifying and standardising bacteriophage plaque morphology measurements. Virology. 2021;561:1–5. doi:10.1016/j.virol.2021.05.011
💡 Only the Published engine mode reproduces this paper byte-for-byte. If you used Current with minor corrections, add "with minor numerical-precision corrections" to your methods.

Statistics — avoid pseudoreplication

⚠️ The plate is the biological replicate, not the individual plaque. Use *_PLATE_MEAN / *_PLATE_SD columns in per_phage.csv, with ≥3 plates per phage. Compare phages with Mann–Whitney or Kruskal–Wallis on the per-plate means, or a mixed model with plate as a random effect.

Required caveats in Methods

Plaque size is reported as the convex-hull area-equivalent diameter (2⋅√(area/π)). This is a slight upper bound that includes any halo — a known property of the published method. State this in your methods.
A tilted phone shot makes the dish elliptical and biases every mm value. Image plates straight-on (orthographic). The tool warns when axis-ratio > 1.03. Report any tilt warnings in supplementary QC.
Apply watershed uniformly across all compared phages, or not at all. State in methods: "Touching plaques were separated using distance-transform watershed segmentation."
Only if you validate them on your own plates. Use the Validate tab to generate precision/recall/F1 scores against your hand-corrected ground truth, and report those numbers. Do not rely on the published Trofimova & Jaschke validation for these in-house modes.
Report OD as the primary turbidity metric. Call it "relative optical density" (not "absorbance") for iPhone/HEIC images; "absorbance" is only accurate for RAW/linear TIFF with dark+flat correction. Always report the reference images you used and whether you had a blank-agar reference.
PFU counts are detected (accepted) plaques only. Validate that they fall in the 30–300 countable range for reliable titer calculations.
