🔬 Plaque Toolkit — how to use, use cases & verification

Measure bacteriophage plaque size, turbidity, count/titer — with an interactive editor, irregular-shape tools, overlap handling, and a validated Fiji/ImageJ bridge. This page shows the workflows with screenshots and the results of verifying the app against hand-labelled ground truth and against real Fiji.

The five-minute workflow

  1. Open a plate photo (TIFF/JPEG/PNG/iPhone HEIC) — drag it in or Open image….
  2. Pick the Engine (★ Precise is the most accurate) and set the Dish size (85 mm agar / 100 mm lid).
  3. The plate auto-detects; the dish is found and the scale is shown (e.g. 0.039 mm/px ✓).
  4. Correct by hand if needed (see tools below), then read the live table or Export all.

Use cases

1 · Measure plaque size (the core)

Every plaque's AREA_MM2 and area-equivalent DIAMETER_MM (2·√(A/π)) appear live; the summary card shows count + median/mean Ø. Sort or Copy (Ctrl+C) into Excel.

2 · Irregular / odd-shaped plaques

Size is always measured from the true outline, not a circle. For comet/blob plaques the auto-tracer misses, use Draw shape (press-drag to outline any shape) or click Add with a higher Trace tol. The CIRCULARITY column (1 = round) flags irregular ones.

3 · Overlapping / confluent plaques

Touching plaques can't be sized reliably, so the app flags them dashed hot-pink and marks OVERLAP=yes. Tick Exclude overlaps from stats to keep them visible but out of the median/mean.

4 · Turbidity & cross-phage comparison

MEAN_GRAY + within-plate TURBIDITY_REL per plaque; the Compare turbidity tab batches a folder of phages into optical-density turbidity, clarity class, count/PFU titer, and figures.

5 · Validate against Fiji / ImageJ

Export ▾ → Fiji registration bundle writes a calibrated crop + an ImageJ ROI set (numbered to match) + a numbered map + a registration CSV. In Fiji, open the crop and load the ROIs (ROI #k = plaque #k), or trace your own and use Compare vs Fiji… to pair by position (handles any rotation/flip) and report per-plaque differences. Full walkthrough in FIJI_TUTORIAL.html.

Sensitive trades precision for recall: it finds ~2–3× more plaques but adds false positives and noisier sizes. Precise is the accurate/conservative choice; Sensitive is the higher-recall choice for dense plates (then prune false positives by eye).

What this means. On these plaques, Precise has perfect precision (no false positives) and its diameters match your hand labels to r≈0.99 / ≤0.0025 mm — the numbers it reports are trustworthy. On very dense plates it is conservative (recall ~0.3–0.4): it under-counts rather than guessing.
Guidance for dense plates. Use Sensitive mode (lower size gates) to catch more, and the Detect area / Add / Draw shape tools to recover the rest by hand — the manual-assist workflow. Enable Split touching plaques for confluent regions.

Honest limitations

  • Precise under-detects on very dense plates (recall ~0.3–0.4 at ~90–185 plaques/plate). Use Sensitive + manual tools there. What it does report is highly accurate.
  • Precise is in-house (not independently/peer-reviewed). Only the Published engine is the peer-reviewed algorithm — cite that for published detection.
  • Turbidity from top-lit phone photos is relative / screening-grade, not calibrated OD.

Ground truth = your hand-labelled plaque sets (Verifications/labels_*.json). Fiji = ImageJ 2.16.0 headless. Reproducible via the verification harness.
